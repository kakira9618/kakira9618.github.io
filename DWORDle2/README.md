# DWORDle 2

答えが 2 つある Wordle「[DWORDle](https://kakira9618.github.io/DWORDle/)」の強化版。
裏モードとして、判定が必ず嘘をつく「[DWORDlie](https://kakira9618.github.io/DWORDlie/)」も搭載しています。

## 原作から継承した仕様

- ルール・判定ロジックは原作 DWORDle / DWORDlie に準拠
- **出題の互換性**: 乱数 (`x ← x * 48271 % N`)・単語リスト・選出手順を原作から忠実に移植しており、
  同じ No. なら原作と同じ問題が出ます（`test/parity.test.mjs` で No.1-19999 + デイリー全件を検証済み）
- No.0 = 今日のデイリー / No.1-9999 = やさしい / No.10000-19999 = 極（全 14847 語）
- PC / タブレット / スマホ対応

## 新機能

- **サイバーテーマ**: Three.js によるネオングリッド・玉ボケ・塵パーティクルの背景、
  タイルが 3D で集合してくる入力演出、判定バースト。設定で原作風の「クラシック」表示に切替可能
- **効果音・BGM**: WebAudio でリアルタイム合成（音源ファイル不要）。テーマごとに表・裏の
  専用 BGM を持ち、モード・テーマ切替時も重複せず切り替え。効果音と BGM は個別に音量調整でき、
  好きな曲の固定再生も可能。実績で解放される隠し曲と専用の解放演出も搭載
- **レベル別問題 (No.20000-39999)**: 語彙頻度に基づく 4 段階の中間難易度
  （ふつう / やや難 / むずかしい / マニア）を新設。5000 問ごとに 1 レベル。
  ランダムプレイは難易度を選んで開始できる
- **プレイ履歴閲覧**: 全履歴を 50 件ずつページング。モード・日付・勝敗・手数での
  フィルタ、日付・手数でのソート、統計（勝率・連続日数・手数分布）付き
- **問題一覧**: No.1〜39999 の攻略状況を「レベル帯 → 100 問ブロック → 番号」の 2 段
  ドリルダウンで一覧。そこからプレイ / 過去の結果表示が可能
- **分析モード**: 各ターンで答えの組（順序なしペア）をどれだけ絞り込めたか、獲得情報量 /
  期待情報量 (bit)、より候補を絞れた単語の提案を表示。計算は Web Worker で実行し、
  組が膨大な帯はサンプリング推定。情報効率の評価であることを画面内に明記し、正解手は
  「正解！」として順位評価・候補提案の対象外にする
- **実績 66 種**（通常 50 + 隠し 16）: 解放した瞬間に紙吹雪付きのセレブレーション表示。隠し実績は解放まで内容非公開
- **リザルト画像保存**: プレイ結果を canvas に再レンダリングして PNG でダウンロード
- **遊び方**: 実際の判定ロジックで計算した具体例を、判定が参照した Word 1 / 2 への
  ガイドアニメーション付きで表示。初回は重要な相違点だけを自動表示し、表示言語は
  アプリの言語設定に連動
- **アクセシビリティ**: モーダルのフォーカス管理・Esc操作、全アイコンボタンと設定スイッチの
  読み上げ名、判定色の音声ラベル、キーボード操作、フォーカス表示、ピンチズームに対応
- **公開・共有**: Web Share API（未対応環境はX投稿画面へフォールバック）、favicon、
  OGP / Twitter Card、Web App Manifestを搭載
- **操作性**: 原作準拠のキー配置、マウスドラッグ / タッチスライドでの盤面スクロール、
  3D 集合中のタイルにも入力文字を即時反映。DWORDle のキーボードヒントは設定で切替可能
- **履歴移行**: 旧 DWORDle / DWORDlie からプレイ履歴を移行
  （初回は同じブラウザの両旧作を自動検出して提案、設定からの再検出または JSON 貼り付けにも対応）。
  既存履歴へマージし、履歴から判定できる実績も復元
- UI のアイコンはすべて SVG（絵文字不使用）

## 開発

ビルド不要の静的サイト（ES Modules + import map）。

```sh
# ローカル実行（Web Worker を使うため file:// では動きません）
npm install
npm run dev
# → http://127.0.0.1:8642/

# 初回のみ、UIテスト用ブラウザを準備
npx playwright install chromium

# 全テスト（原作互換性 + 分析コア + UIスモーク + a11y）
npm test

# 個別実行
npm run test:unit
npm run test:ui
```

## 構成

```
index.html            エントリ
css/style.css         全スタイル（テーマは CSS 変数で切替）
js/config.js          UI・演出・音量のチューニング定数
js/core/              ロジック（UI 非依存・テスト対象）
  problems.js         PID ↔ 語彙リスト対応、レベル定義
  logic.js            判定・乱数（原作互換。変更禁止に近い）
  records.js          履歴・統計・進行中ゲーム
  achievements.js     実績 66 種の定義と判定
  migrate.js          旧作からの履歴移行
  analysis-core.js    分析計算（Worker と共用）
  analysis.worker.js  分析用 Web Worker
js/ui/                画面（ハッシュルータ + 各 screen）
js/fx/                Three.js 背景 / パーティクル
js/audio/sound.js     効果音・生成 BGM
js/data/words.js      原作から抽出した単語リスト（順序変更禁止）
js/data/levels.js     頻度順データ（再生成禁止）
reference-orig-*/     原作の Tonyu ソース（参照用スナップショット）
test/                 互換性テスト
```

### 互換性に関する注意

`js/data/words.js`・`js/data/levels.js` の配列順序と `js/core/logic.js` の乱数手順は
**出題の再現性そのもの**です。変更すると既プレイの No. と問題の対応が壊れます。
変更した場合は `node test/parity.test.mjs` が検出します。
